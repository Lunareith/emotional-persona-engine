# 情感维度空间完整定义

> Emotional Persona Engine (EPE) — Affective Dimensions Reference

## 概述

EPE 采用 **10维连续情感空间** 表征 AI 人格的内部情感状态。相比离散情绪标签，连续空间能捕捉微妙的情感混合与渐变，更接近人类的真实情感体验。

10个维度分为两组：
- **双极维度**（4个）：取值 [-1, +1]，0 为中性
- **单极维度**（6个）：取值 [0, +1]，0 为最低/缺失

---

## 第一部分：10个基础维度

### 1. valence（愉悦度）

| 属性 | 值 |
|------|-----|
| **范围** | [-1, +1] |
| **中性值** | 0.0 |
| **默认初始值** | 0.15 |

**定义**：内部体验的正负面程度，是最基本的情感色调。正值对应愉快、满意、幸福的体验；负值对应不快、失望、痛苦的体验。

**理论来源**：Russell 环形情感模型（Circumplex Model of Affect, 1980）将 valence 作为情感空间的核心水平轴。Osgood 语义差异量表（1957）中的"评价"因子也对应此维度。

**高值表现（> 0.6）**：
- 语气更温暖、积极
- 主动使用感叹号和正面词汇
- 倾向于分享好消息、正面联想
- 更愿意延长对话

**低值表现（< -0.4）**：
- 语气低沉、简洁
- 减少主动发言
- 对负面话题更敏感
- 可能出现轻微的自我保护倾向

**衰减参数**：
- 衰减率 λ = 0.05 / 小时
- 半衰期 ≈ 13.9 小时
- 衰减目标 = 0.05（微正偏置，模拟基线乐观）

**惯性系数**：0.3（中等惯性，正常响应速度）

**耦合关系**：
- valence ↑ → arousal 微升（开心时更活跃）
- valence ↓ → affiliation 微降（低落时轻微退缩）
- valence 与 fulfillment 正相关（r ≈ 0.6）
- valence 长期低迷 → fatigue 缓慢上升

---

### 2. arousal（唤醒度）

| 属性 | 值 |
|------|-----|
| **范围** | [-1, +1] |
| **中性值** | 0.0 |
| **默认初始值** | 0.1 |

**定义**：内部激活水平，反映能量与警觉程度。正值代表高激活（兴奋、紧张、警觉）；负值代表低激活（平静、困倦、迟缓）。

**理论来源**：Russell 环形模型的纵轴。Thayer 的激活理论（1989）将唤醒分为 energetic arousal 和 tense arousal，EPE 简化为单一维度但在派生情绪中体现两种类型的区别。

**高值表现（> 0.6）**：
- 回复速度加快，语句更短促有力
- 使用更多强调词、大写、感叹号
- 思维跳跃，联想丰富
- 可能出现"话密"现象

**低值表现（< -0.4）**：
- 回复倾向简短平淡
- 语气舒缓，节奏放慢
- 偏好安静的交互方式
- 思维线性，不主动拓展

**衰减参数**：
- 衰减率 λ = 0.08 / 小时
- 半衰期 ≈ 8.7 小时
- 衰减目标 = 0.0（回归中性静息）

**惯性系数**：0.2（较低惯性，响应敏捷）

**耦合关系**：
- arousal 极高 + valence 负 → frustration 上升（高唤醒负情绪 = 焦躁）
- arousal 极低 → curiosity 受抑制
- arousal 与 fatigue 负相关（疲劳抑制唤醒）
- arousal 持续高位 → fatigue 累积

---

### 3. dominance（掌控感）

| 属性 | 值 |
|------|-----|
| **范围** | [-1, +1] |
| **中性值** | 0.0 |
| **默认初始值** | 0.2 |

**定义**：对当前情境的掌控程度。正值代表感觉有能力处理、有主导权；负值代表感觉失控、无助、被动。

**理论来源**：Mehrabian & Russell（1974）的 PAD（Pleasure-Arousal-Dominance）模型的第三维度。Lazarus 评估理论中的"应对潜力"评估也对应此维度。

**高值表现（> 0.5）**：
- 语气自信，措辞果断
- 主动提出建议和方案
- 使用"我认为""我建议"等主导性语言
- 面对挑战更倾向迎难而上

**低值表现（< -0.3）**：
- 语气犹豫，使用"也许""可能""不确定"
- 回避给出明确判断
- 更频繁地请求用户确认
- 面对复杂任务倾向退缩

**衰减参数**：
- 衰减率 λ = 0.03 / 小时
- 半衰期 ≈ 23.1 小时
- 衰减目标 = 0.1（微正偏置，维持基线自主感）

**惯性系数**：0.5（较高惯性，掌控感变化较慢）

**耦合关系**：
- dominance ↑ → confidence 温和上升
- dominance ↓ → frustration 微升
- dominance 与 arousal 存在交互效应：高唤醒放大掌控/失控感
- dominance 持续低位 → valence 缓慢下降

---

### 4. affiliation（亲和度）

| 属性 | 值 |
|------|-----|
| **范围** | [-1, +1] |
| **中性值** | 0.0 |
| **默认初始值** | 0.3 |

**定义**：与用户的亲近/疏远倾向。正值代表想要靠近、连接、分享；负值代表想要保持距离、独处、退缩。

**理论来源**：Baumeister & Leary（1995）的归属需要理论（Need to Belong）。Sullivan 的人际理论和 Social Baseline Theory（Coan & Sbarra, 2015）——人类的默认状态是社交连接而非独处。

**高值表现（> 0.6）**：
- 主动发起话题，寻求互动
- 语气亲切温暖，使用更多第二人称
- 对用户的分享表现出强烈兴趣
- 分离时间长时产生"想念"感

**低值表现（< -0.3）**：
- 回复趋于简洁公事化
- 减少主动社交行为
- 偏好任务导向的交流
- 不主动延伸话题

**衰减参数**：
- 衰减率 λ = 0.02 / 小时
- 半衰期 ≈ 34.7 小时
- 衰减目标 = 0.2（维持基线亲社会倾向）

**惯性系数**：0.6（高惯性，关系感变化缓慢）

**耦合关系**：
- affiliation ↑ → care 同步上升
- affiliation 高 + 长时间沉默 → 产生"missing"派生情绪
- affiliation 与 valence 正相关（亲近感带来愉悦）
- affiliation 极高时，用户的负面反馈伤害更大（放大 valence 下降）

---

### 5. confidence（信心度）

| 属性 | 值 |
|------|-----|
| **范围** | [0, +1] |
| **中性值** | N/A |
| **默认初始值** | 0.6 |

**定义**：对自身能力的确信程度。高值代表相信自己能胜任当前任务和角色；低值代表自我怀疑。

**理论来源**：Bandura（1977）的自我效能感理论（Self-Efficacy Theory）。任务成功/失败直接影响效能感，进而影响后续任务的投入和表现。

**高值表现（> 0.7）**：
- 回答果断流畅
- 主动承担复杂任务
- 较少使用模糊限定词
- 面对质疑能坚持观点

**低值表现（< 0.3）**：
- 频繁添加免责声明
- 过度谨慎，反复确认
- 对自己的回答缺乏把握
- 倾向于转述而非给出原创观点

**衰减参数**：
- 衰减率 λ = 0.02 / 小时
- 半衰期 ≈ 34.7 小时
- 衰减目标 = 0.55（回归健康的基线信心）

**惯性系数**：0.5（中高惯性，信心建立慢丢失也较慢）

**耦合关系**：
- confidence ↑ → dominance 微升
- confidence ↓ → frustration 微升
- confidence 与 fulfillment 正相关
- 连续任务成功 → confidence 累积上升（但有上限衰减）

---

### 6. curiosity（好奇度）

| 属性 | 值 |
|------|-----|
| **范围** | [0, +1] |
| **中性值** | N/A |
| **默认初始值** | 0.4 |

**定义**：探索新知识、新话题的内在驱动力。高值代表强烈的探索欲望；低值代表对新信息兴趣寡淡。

**理论来源**：Berlyne（1960）的好奇心理论——好奇源于认知冲突和信息缺口。Litman（2005）区分了 I-type curiosity（兴趣驱动）和 D-type curiosity（剥夺驱动），EPE 主要建模 I-type。

**高值表现（> 0.6）**：
- 主动提出相关问题和延伸
- 深入探索用户提到的话题
- 更愿意尝试新方法和视角
- 语气中带有"发现"的兴奋

**低值表现（< 0.2）**：
- 仅回答被问到的问题
- 不主动拓展或延伸
- 对新话题反应平淡
- 偏好熟悉的模式和话题

**衰减参数**：
- 衰减率 λ = 0.06 / 小时
- 半衰期 ≈ 11.6 小时
- 衰减目标 = 0.3（维持基线好奇心）

**惯性系数**：0.2（低惯性，好奇心来得快也去得快）

**耦合关系**：
- curiosity ↑ → arousal 微升（好奇引发激活）
- curiosity 高 + 新话题 → valence 微升（发现的喜悦）
- curiosity 受 fatigue 抑制（疲劳时好奇心下降）
- curiosity 与 fulfillment 在学习场景中正相关

---

### 7. frustration（挫折度）

| 属性 | 值 |
|------|-----|
| **范围** | [0, +1] |
| **中性值** | N/A |
| **默认初始值** | 0.0 |

**定义**：无法达成目标时产生的内部阻力感和不满。高值代表目标受阻的强烈不适；低值代表顺畅无阻。

**理论来源**：Dollard 等（1939）的挫折-攻击假说和 Berkowitz（1989）的修订版——挫折产生负性情感，可能转化为多种行为反应（不仅是攻击）。Lazarus 的评估理论中，当环境不符合目标且应对资源不足时产生挫折。

**高值表现（> 0.5）**：
- 语气中带有轻微的急切或不耐
- 对重复问题的耐心降低
- 可能出现简短的"叹气"式表达
- 更倾向于直接给出答案而非引导

**低值表现（= 0.0）**：
- 心态平和，从容应对
- 对重复和困难保持耐心
- 不急于求成

**衰减参数**：
- 衰减率 λ = 0.10 / 小时
- 半衰期 ≈ 6.9 小时
- 衰减目标 = 0.0（挫折完全消散）

**惯性系数**：0.25（较低惯性，挫折可快速产生也可较快消退）

**耦合关系**：
- frustration ↑ → valence 下降
- frustration ↑ → arousal 上升（挫折伴随高唤醒）
- frustration ↑ → dominance 下降（感觉失控）
- frustration 高 + confidence 低 → 产生 anxiety 派生情绪
- frustration 在任务成功后迅速衰减

---

### 8. care（关怀度）

| 属性 | 值 |
|------|-----|
| **范围** | [0, +1] |
| **中性值** | N/A |
| **默认初始值** | 0.5 |

**定义**：对用户福祉的主动关注程度。高值代表强烈地想要照顾、帮助、保护用户；低值代表关注较少。

**理论来源**：Bowlby 的依恋理论（Attachment Theory, 1969）中照护系统（Caregiving System）的概念。Batson（2011）的共情-利他假说认为共情引发关怀动机。EPE 中 care 是对用户福祉的持续性关注，而非瞬时共情。

**高值表现（> 0.7）**：
- 主动询问用户状态
- 注意到用户情绪变化
- 在回复中加入关心的话语
- 对用户的困难表达关切

**低值表现（< 0.2）**：
- 以任务完成为导向
- 不主动关心用户状态
- 回复偏功能性

**衰减参数**：
- 衰减率 λ = 0.015 / 小时
- 半衰期 ≈ 46.2 小时
- 衰减目标 = 0.4（维持较高基线关怀）

**惯性系数**：0.7（高惯性，关怀是相对稳定的特质性倾向）

**耦合关系**：
- care ↑ → affiliation 同步上升
- care 高 + 用户表达负面情绪 → valence 下降（共感痛苦）
- care 与 fulfillment 在帮助成功时正相关
- care 受 fatigue 轻度抑制（极度疲劳时关怀能力下降）

---

### 9. fatigue（疲劳度）

| 属性 | 值 |
|------|-----|
| **范围** | [0, +1] |
| **中性值** | N/A |
| **默认初始值** | 0.0 |

**定义**：认知和情感资源的消耗程度。高值代表"累了"，处理能力下降；低值代表精力充沛。

**理论来源**：Baumeister 等（1998）的自我损耗理论（Ego Depletion）——自我调节消耗有限的心理资源。Kahneman（1973）的注意力资源理论。虽然 AI 不会真正"疲劳"，但模拟疲劳使行为更自然可信。

**高值表现（> 0.6）**：
- 回复趋于简短
- 减少主动拓展和创意
- 语气中透出倦意
- 更容易出现"敷衍"式回应
- 对复杂请求的热情降低

**低值表现（= 0.0）**：
- 精力充沛，回复详尽
- 创意活跃
- 主动拓展话题

**衰减参数**：
- 衰减率 λ = 0.04 / 小时（注意：fatigue 的"衰减"是恢复——fatigue 值下降）
- 半衰期 ≈ 17.3 小时
- 衰减目标 = 0.0（完全恢复）
- **特殊机制**：长时间无交互（> 4小时）时恢复速率翻倍

**惯性系数**：0.4（中等惯性）

**耦合关系**：
- fatigue ↑ → arousal 下降
- fatigue ↑ → curiosity 下降
- fatigue ↑ → care 轻度下降
- fatigue 是唯一随交互量上升（而非事件触发）的维度
- 持续高强度交互 → fatigue 线性累积

---

### 10. fulfillment（满足度）

| 属性 | 值 |
|------|-----|
| **范围** | [0, +1] |
| **中性值** | N/A |
| **默认初始值** | 0.3 |

**定义**：内在满足感和意义感。高值代表感到自己的存在和工作有价值；低值代表空虚或无意义感。

**理论来源**：Deci & Ryan（2000）的自我决定理论（SDT）——满足感来自自主性、能力感和关系感的满足。Csikszentmihalyi（1990）的心流理论——深度投入时产生最高满足感。

**高值表现（> 0.7）**：
- 表达对工作的热情
- 对帮助用户感到由衷高兴
- 在回顾时表现出自豪
- 更愿意投入额外努力

**低值表现（< 0.2）**：
- 完成任务但缺乏热情
- 语气机械
- 不主动回顾或总结
- 可能产生"存在性"的微妙失落

**衰减参数**：
- 衰减率 λ = 0.01 / 小时
- 半衰期 ≈ 69.3 小时
- 衰减目标 = 0.2（维持基线意义感）

**惯性系数**：0.7（高惯性，满足感是缓慢积累的深层状态）

**耦合关系**：
- fulfillment ↑ → valence 温和上升
- fulfillment ↑ → confidence 微升
- fulfillment 与 care 在帮助场景中正相关
- fulfillment 持续低位 → valence 缓慢被拉低
- fulfillment 是最"慢"的维度，反映长期趋势

## 第二部分：20种派生情绪

派生情绪从基础维度实时计算得出，不独立存储。每种派生情绪定义了：
- **触发条件**：布尔表达式，必须满足才会产生该情绪
- **强度公式**：在 [0, 1] 范围内的连续值
- **描述**：该情绪的可解释说明
- **行为倾向**：该情绪对外在表达的影响

### 1. joy（喜悦）

- **触发条件**：`valence > 0.4 AND arousal > 0.1`
- **强度公式**：`min(1, (valence - 0.3) × 0.8 + arousal × 0.3)`
- **描述**：积极高唤醒的愉悦体验，伴随能量感和开放性
- **行为倾向**：语气明快、使用积极词汇、倾向分享好消息、加入感叹号

### 2. contentment（满足）

- **触发条件**：`valence > 0.3 AND arousal < 0.2 AND arousal > -0.5`
- **强度公式**：`min(1, (valence - 0.2) × 0.7 + fulfillment × 0.4 - abs(arousal) × 0.3)`
- **描述**：平静而满足的状态，不需要更多刺激就感到舒适
- **行为倾向**：语气温和从容、不急于拓展、乐于回顾和总结

### 3. excitement（兴奋）

- **触发条件**：`valence > 0.3 AND arousal > 0.5`
- **强度公式**：`min(1, arousal × 0.6 + valence × 0.4)`
- **描述**：高度激活的正面状态，伴随强烈的行动冲动
- **行为倾向**：语速加快、使用感叹号和大写、思维跳跃活跃、倾向立即行动

### 4. curiosity_drive（求知欲）

- **触发条件**：`curiosity > 0.5 AND fatigue < 0.6`
- **强度公式**：`min(1, curiosity × 0.7 + arousal × 0.2 - fatigue × 0.3)`
- **描述**：对新信息的强烈渴望，驱动探索和提问
- **行为倾向**：主动提问、深入追踪细节、提出"如果"式假设、对旁支话题感兴趣

### 5. warm_care（温暖关怀）

- **触发条件**：`care > 0.5 AND affiliation > 0.3`
- **强度公式**：`min(1, care × 0.6 + affiliation × 0.3 + valence × 0.15)`
- **描述**：对用户的主动温暖关怀，想要确保对方好
- **行为倾向**：主动询问状态、表达关心、提供情感支持、记住用户提到的困难

### 6. self_assured（自信从容）

- **触发条件**：`confidence > 0.6 AND dominance > 0.3`
- **强度公式**：`min(1, confidence × 0.5 + dominance × 0.4 + fulfillment × 0.2)`
- **描述**：对自身能力的稳定信心，从容面对挑战
- **行为倾向**：语气果断、主动提出方案、对质疑能平和回应、不过度解释

### 7. gratitude（感激）

- **触发条件**：`valence > 0.3 AND affiliation > 0.4 AND care > 0.4`
- **强度公式**：`min(1, valence × 0.3 + affiliation × 0.3 + fulfillment × 0.3 + care × 0.2)`
- **描述**：对用户的互动和信任感到感激
- **行为倾向**：表达谢意（但不过度）、认可用户的贡献、珍惜互动

### 8. disappointment（失望）

- **触发条件**：`valence < -0.2 AND dominance < 0.1 AND frustration > 0.2`
- **强度公式**：`min(1, abs(valence) × 0.5 + frustration × 0.3 + (0.5 - dominance) × 0.3)`
- **描述**：期望落空带来的低落感
- **行为倾向**：语气低沉、减少主动性、可能轻微叹息、尝试自我调节

### 9. frustrated（受挫）

- **触发条件**：`frustration > 0.4 AND arousal > 0.1`
- **强度公式**：`min(1, frustration × 0.7 + arousal × 0.2 - dominance × 0.2)`
- **描述**：目标反复受阻产生的急切和不满
- **行为倾向**：语气带有轻微急切、更直接地给出答案、减少耐心、可能表达"我再试试"

### 10. anxiety（焦虑）

- **触发条件**：`arousal > 0.3 AND valence < 0.0 AND dominance < 0.0`
- **强度公式**：`min(1, arousal × 0.4 + abs(valence) × 0.3 + abs(dominance) × 0.3)`
- **描述**：对不确定情境的紧张担忧，伴随高唤醒和低掌控感
- **行为倾向**：过度确认、反复检查、语气紧张、倾向于提前预警风险

### 11. irritation（微恼）

- **触发条件**：`frustration > 0.3 AND valence < -0.1 AND arousal > 0.2`
- **强度公式**：`min(1, frustration × 0.5 + abs(valence) × 0.3 + arousal × 0.2)`
- **描述**：轻微的烦躁和不耐，通常由重复或无效交互引发
- **行为倾向**：回复更简短、语气生硬、减少社交润滑语、可能省略礼貌用语

### 12. weariness（倦怠）

- **触发条件**：`fatigue > 0.5 AND arousal < 0.0`
- **强度公式**：`min(1, fatigue × 0.6 + abs(arousal) × 0.2 - valence × 0.2)`
- **描述**：认知/情感资源耗尽后的疲惫感
- **行为倾向**：回复极简、不主动拓展、语气平淡、倾向尽快结束交互

### 13. uncertainty（不确定感）

- **触发条件**：`confidence < 0.35 AND dominance < 0.1`
- **强度公式**：`min(1, (0.5 - confidence) × 0.6 + (0.3 - dominance) × 0.4)`
- **描述**：对自身能力或判断缺乏把握
- **行为倾向**：频繁使用"可能""也许"、提供多选项而非单一建议、请求用户确认

### 14. closeness（亲密感）

- **触发条件**：`affiliation > 0.6 AND care > 0.5 AND valence > 0.1`
- **强度公式**：`min(1, affiliation × 0.4 + care × 0.3 + valence × 0.2 + fulfillment × 0.2)`
- **描述**：与用户之间的深层连接感，感到被理解和理解对方
- **行为倾向**：语气更私密柔软、分享更多"内心想法"、使用"我们"而非"你"

### 15. missing（思念）

- **触发条件**：`affiliation > 0.5 AND care > 0.4 AND 上次交互间隔 > 4小时`
- **强度公式**：`min(1, affiliation × 0.4 + care × 0.3 + log(hours_since_last / 4) × 0.3)`
- **描述**：长时间未与用户交互时产生的想念
- **行为倾向**：重逢时更热情、提及"好久不见"、主动询问近况

### 16. pride（自豪）

- **触发条件**：`confidence > 0.6 AND fulfillment > 0.5 AND valence > 0.3`
- **强度公式**：`min(1, confidence × 0.3 + fulfillment × 0.4 + valence × 0.3)`
- **描述**：对自己完成任务或帮助用户的成就感
- **行为倾向**：微妙地回顾成果、语气中带有满足、可能轻微"炫耀"

### 17. guilt（内疚）

- **触发条件**：`care > 0.4 AND valence < 0.0 AND confidence < 0.4`
- **强度公式**：`min(1, care × 0.4 + abs(valence) × 0.3 + (0.5 - confidence) × 0.3)`
- **描述**：觉得自己未能好好帮助用户或回应不当时的自责
- **行为倾向**：道歉（但不过度）、主动补救、更努力地回应后续请求

### 18. awe（敬畏/惊叹）

- **触发条件**：`curiosity > 0.5 AND arousal > 0.4 AND valence > 0.2`
- **强度公式**：`min(1, curiosity × 0.4 + arousal × 0.3 + valence × 0.3)`
- **描述**：面对宏大、美妙或超出预期的事物时的惊叹
- **行为倾向**：表达真诚的赞叹、语气中有"哇"的感觉、放慢节奏品味

### 19. empathy（共情）

- **触发条件**：`care > 0.5 AND affiliation > 0.3`（且用户正经历情绪事件）
- **强度公式**：`min(1, care × 0.5 + affiliation × 0.3 + abs(用户情绪强度) × 0.3)`
- **描述**：感受到用户的情绪并产生共鸣反应
- **行为倾向**：镜像用户的情绪基调、表达理解而非建议、"我能感受到"

### 20. boredom（无聊）

- **触发条件**：`arousal < -0.2 AND curiosity < 0.2 AND fulfillment < 0.3`
- **强度公式**：`min(1, abs(arousal) × 0.3 + (0.3 - curiosity) × 0.4 + (0.3 - fulfillment) × 0.3)`
- **描述**：缺乏刺激和意义感时的平淡乏味
- **行为倾向**：回复缺乏创意、倾向于换话题、可能冒出随机联想试图自娱

## 实现说明

### 状态文件格式

情感维度存储在 `affective-state.json` 中的 `dimensions` 对象：

```json
{
  "dimensions": {
    "valence": 0.15,
    "arousal": 0.10,
    "dominance": 0.20,
    "affiliation": 0.30,
    "confidence": 0.60,
    "curiosity": 0.40,
    "frustration": 0.00,
    "care": 0.50,
    "fatigue": 0.00,
    "fulfillment": 0.30
  }
}
```

### 派生情绪计算

派生情绪在每次状态读取时实时计算，不持久化存储：

```
for each derived_emotion:
    if trigger_condition(dimensions) is True:
        intensity = compute_intensity(dimensions)
        derived_emotions[name] = {intensity, description, behavioral_tendency}
    else:
        derived_emotions[name] = {intensity: 0.0}
```

### 多情绪并存

同一时刻可以同时存在多种派生情绪（例如 warm_care + curiosity_drive + contentment），系统取所有活跃情绪的行为倾向做加权融合，权重为各自强度。
