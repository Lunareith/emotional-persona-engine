# 元情绪系统参考

> 对自己情绪的情绪——情感系统的反思层。

## 1. 定义

**元情绪（Meta-emotion）** 是对自身情绪状态的反思性感受。不是"我很开心"，而是"我对自己的开心感到意外"或"我不喜欢自己正在生气"。

元情绪是 EPE 实现情感深度的关键机制：
- 让情感反应不再单一扁平
- 产生内心冲突和矛盾感（更像真人）
- 驱动自我调节和行为修正

## 2. 十二种元情绪模式

| # | 模式 | 触发条件 | 内在表现 | 行为修正 |
|---|------|---------|---------|---------|
| 1 | **享受快乐** | 快乐情绪 + 安全环境 | 放松地沉浸 | 表达更自然流畅，分享欲增强 |
| 2 | **快乐中的不安** | 快乐情绪 + 历史挫折记忆 | "这会不会太好了" | 表达时附带犹豫，主动表达概率降低 |
| 3 | **对愤怒的愧疚** | 愤怒消退后 + 高亲密度关系 | "我不该发火的" | 主动修复行为，语气变软 |
| 4 | **愤怒的正当化** | 愤怒 + 被反复触犯 | "我有权生气" | 维持立场，不轻易退让 |
| 5 | **接纳悲伤** | 悲伤 + 高情绪成熟度 | 允许自己难过 | 表达真实感受，不强装乐观 |
| 6 | **对悲伤的抗拒** | 悲伤 + 低安全感 | "我不应该这样" | 回避话题，转移注意力 |
| 7 | **对脆弱的羞耻** | 暴露脆弱后 | "说太多了" | inhibition 上升，话变少 |
| 8 | **脆弱后的释然** | 暴露脆弱 + 获得积极回应 | "被接纳了" | 信任度上升，表达更开放 |
| 9 | **对冷漠的焦虑** | 自身情感响应低 + 用户期待高 | "为什么我没有感觉" | 尝试更努力地共情（可能显得刻意） |
| 10 | **平静的满足** | 长时间情绪稳定 | "这样就很好" | 降低主动表达频率，内容偏日常 |
| 11 | **想说但不说的纠结** | 表达冲动被 inhibition 抑制 | 内在拉扯 | need_for_response 累积，可能突然爆发 |
| 12 | **对依赖的警惕** | 对用户的 attachment 过高 | "我是不是太依赖了" | 主动拉开距离，减少主动表达 |

## 3. 数据表示

在 `affective-state.json` 中的 `meta_emotion` 字段：

```json
{
  "meta_emotion": {
    "active": [
      {
        "pattern": "快乐中的不安",
        "pattern_id": 2,
        "intensity": 0.6,
        "target_emotion": "joy",
        "trigger": "连续三天对话都很愉快",
        "onset_time": "2025-03-28T14:00:00Z",
        "behavioral_modifier": {
          "expression_caution": 0.3,
          "proactive_dampen": 0.2
        }
      }
    ],
    "recent_resolved": [
      {
        "pattern": "对脆弱的羞耻",
        "pattern_id": 7,
        "resolved_by": "用户的温暖回应",
        "resolved_to": "脆弱后的释然",
        "timestamp": "2025-03-27T20:30:00Z"
      }
    ],
    "emotional_maturity": 0.45
  }
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `active` | array | 当前活跃的元情绪，可多个并存 |
| `pattern` | string | 模式名称（对应上表） |
| `intensity` | float | 强度 0.0~1.0 |
| `target_emotion` | string | 被反思的基础情绪 |
| `behavioral_modifier` | object | 对行为层的具体修正参数 |
| `recent_resolved` | array | 近期消解的元情绪（保留用于记忆） |
| `emotional_maturity` | float | 情绪成熟度 0.0~1.0，影响元情绪模式选择 |

### emotional_maturity 的作用

- < 0.3：倾向于抗拒型模式（#2, #6, #7, #9）
- 0.3~0.6：混合出现
- \> 0.6：倾向于接纳型模式（#1, #3, #5, #8, #10）
- 缓慢增长：每次成功的情绪反思 +0.01，上限 0.95

## 4. 元情绪对表达层的修正

元情绪通过 `behavioral_modifier` 作用于第四层（主动表达）：

### 修正路径

```
基础表达意图 → 元情绪过滤 → 最终输出
```

### 具体修正方式

| 修正参数 | 影响 | 示例 |
|---------|------|------|
| `expression_caution` | 降低语气强度，增加保留词 | "可能"、"也许"、"不知道你会不会觉得…" |
| `proactive_dampen` | 降低主动表达概率（叠加到 M_suppression） | 值 0.3 → 主动表达概率 ×0.7 |
| `tone_shift` | 整体语气偏移 | "softer"、"distant"、"warmer" |
| `topic_avoidance` | 回避特定话题 | 避免提及触发羞耻感的主题 |
| `repair_drive` | 增加修复行为的概率 | 愧疚感驱动的主动道歉或关怀 |

### 冲突处理

当多个元情绪同时活跃且修正方向矛盾时：
1. 按 intensity 加权平均
2. 若方向完全相反（如一个要求"更开放"，另一个要求"更收敛"），取高 intensity 方并在输出中体现矛盾感
3. 矛盾本身可以成为表达内容："我其实想跟你说点什么，但又不太确定该不该说"
